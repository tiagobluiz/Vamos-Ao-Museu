## Repository Initialization

### 2026-06-14 - Start With Minimal Monorepo Skeleton

Decided to initialize the repository with a minimal monorepo skeleton before selecting frontend, backend, or database technologies.

Why: the product direction includes map discovery, rich filtering, events, badges, leaderboard, and user profiles, but several architectural choices are still open. A lightweight structure gives the project a clear place for code and documentation without forcing premature framework or database commitments.

Rejected for now:

- Full frontend/backend framework scaffolding, because the stack has not been selected yet.
- Choosing SQL or document database immediately, because the data model and query needs still need comparison.
- Expanding or rewriting existing design material, because repository initialization does not require changing it.
